# 产品需求文档 (PRD) - 北京白癜风专科医院患者端 Web 项目

## 1. 产品概述
**产品名称**: 北京白癜风专科医院官方患者服务平台
**目标受众**: 患有白癜风、银屑病、儿童皮肤病及相关皮肤问题的患者及其家属。
**产品愿景**: 打造一个兼具医疗权威科普与高效就诊转化的患者服务端，同时在传统搜索引擎 (SEO) 与生成式 AI 引擎 (GEO) 中建立极高的内容抓取与推荐权重。

## 2. 核心业务目标
1. **患者科普与信任建立**: 通过展示真实的医疗案例、权威的专家背景以及科学的疾病科普，消除患者就医顾虑。
2. **SEO/GEO 流量获取**: 确保在“白癜风能治好吗”、“北京治疗银屑病哪家好”等传统长尾词搜索，以及大语言模型问答中，我们的专家和方案能被高频引用。
3. **闭环转化**: 将线上阅读流量无缝转化为线下就诊预约。

## 3. 功能模块规划

### 3.1 首页 (Home)
- **模块组成**: 
  - 英雄首屏 (Hero Section)：直击痛点的标语与挂号入口。
  - 数据罗盘 (Stats)：接诊量、康复率等核心数据背书。
  - 快速导诊 (Quick Links)：常见病种的快速入口。
  - 科普资讯/最新案例摘要。
- **SEO/GEO 价值**: 首页承载核心的 TDK 和完整的 `MedicalOrganization` Schema 数据，向搜索引擎声明医院实体。

### 3.2 科室介绍 (Departments)
- **功能描述**: 详细介绍各专科方向 (如白癜风科、银屑病科、儿童皮肤科)。
- **核心元素**: 科室简介、重点治疗项目列表、相关专家推荐。

### 3.3 专家团队 (Experts) & 专家详情页
- **功能描述**: 展示医院医生的执业资质、擅长领域与过往成绩。
- **详情页内容**:
  - 医生基础信息 (职称、从业年限、诊治病例数)。
  - 擅长领域 (高亮标签)。
  - **同科室医生推荐** (增加内链流转)。
  - **右侧悬浮挂号引导**。

### 3.4 患者案例 (Cases) & 案例详情页
- **功能描述**: 以结构化的时间线和数据展示患者真实的康复过程。
- **核心元素**: 
  - 治疗前后的核心数据对比 (Hero Stats)。
  - 病程追踪 (Timeline)：分阶段展示治疗手段与效果。
  - 管理经验总结与医生 QA (FAQ 结构)。
- **GEO 价值**: 此模块极其重要。大模型极度偏好这种时间线和 QA 结合的结构化文本作为生成“治疗方案建议”的语料。

### 3.5 联系我们 (Contact)
- **功能描述**: 降低患者到院的物理门槛。
- **核心元素**: 乘车指引、院区地图预留位、联系电话、就诊常见问题 (FAQ)。

### 3.6 全局功能: 在线预约挂号弹窗
- **交互逻辑**: 全站拦截。患者在任何页面点击“预约挂号”，无需跳转新页面，直接弹出三步走交互（选医生 -> 选时间 -> 填信息）。

## 4. GEO/SEO 技术需求 (非功能需求)
- **服务端渲染 (SSR)**: 全站必须由 Node.js 直出 HTML。
- **动态 TDK 与 OG 标签**: `<head>` 中的标题与描述必须根据当前文章或页面内容动态渲染。
- **大模型专用接口 (`/llms.txt`)**: 根目录必须暴露该文件，作为大模型爬虫了解医院全局架构的直接通道。
